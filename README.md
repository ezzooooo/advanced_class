# 🚀 Advanced반 소개

## 한 줄 요약

> **"앱을 만드는 개발자"에서 "실무 프로세스를 아는 개발자"로**

---

## 🤔 왜 Advanced반이 필요한가요?

앱을 "만들 수 있는 것"과 "실무처럼 관리할 수 있는 것"은 다릅니다.

| 비유       | 기본           | 심화                            |
| ---------- | -------------- | ------------------------------- |
| 🚗 자동차  | 운전하는 법    | 정비, 보험, 사고 대응           |
| 🍳 요리    | 음식 만드는 법 | 위생 관리, 원가 계산, 손님 응대 |
| 📱 앱 개발 | 앱 만드는 법   | 테스팅, 자동화, 품질 관리       |

**Advanced반**에서는 앱을 "실무처럼 관리할 수 있게" 됩니다.

---

## 📊 실무에서 일어나는 일들

실제 회사에서 앱 개발자가 하는 일을 생각해보세요:

```
✅ 코드 작성
   ↓
❓ 내 코드가 버그 없이 잘 동작하는지 어떻게 확인하지?  → 테스팅
   ↓
❓ 매번 수동으로 빌드하고 테스트하기 너무 귀찮은데?  → CI/CD
   ↓
❓ 앱 스토어에 올리려면 뭘 준비해야 하지?          → 배포 준비
   ↓
❓ 사용자들이 앱을 안 쓰는데 어떻게 다시 부르지?   → Push 알림
   ↓
❓ 앱 업데이트 없이 설정을 바꾸고 싶은데?         → Remote Config
   ↓
❓ 해커가 우리 앱 데이터를 훔쳐가면 어떡하지?     → 보안
```

**이 모든 것을 Advanced반에서 배웁니다.**

## 💼 취업할 때 이점

### 면접에서 자신있게 답변할 수 있어요

| 질문                          | 수료 후 답변                            |
| ----------------------------- | --------------------------------------- |
| "테스트 코드 작성해보셨나요?" | "Unit Test, Widget Test 작성해봤습니다" |
| "CI/CD 경험 있으세요?"        | "GitHub Actions로 구축해봤습니다"       |
| "배포 프로세스 아세요?"       | "빌드, 서명, 배포 과정을 알아요"        |
| "Remote Config 써보셨나요?"   | "Feature Flag로 기능 제어해봤습니다"    |

### 포트폴리오가 달라져요

```
📱 Advanced반 수료생 포트폴리오
├── 앱 소스코드
├── 테스트 코드 ⭐
├── CI/CD 파이프라인 (.github/workflows) ⭐
├── Remote Config / Feature Flag 적용 ⭐
└── README.md
```

신입 개발자 중에서 이런 경험 있는 분이 드뭅니다. **차별화된 경쟁력**이 됩니다.

## 📅 7주 동안 배우는 것

| 주차  | 주제              | 이런 걸 할 수 있게 돼요                 |
| :---: | ----------------- | --------------------------------------- |
| 1주차 | **테스팅**        | 내 코드가 잘 동작하는지 자동으로 확인   |
| 2주차 | **성능 최적화**   | 앱이 버벅거릴 때 원인 찾고 해결         |
| 3주차 | **CI/CD**         | 코드 푸시하면 자동으로 테스트/빌드      |
| 4주차 | **앱 배포 준비**  | 스토어 배포를 위한 빌드, 서명 과정 이해 |
| 5주차 | **Push 알림**     | 사용자에게 알림 보내기                  |
| 6주차 | **Remote Config** | 앱 업데이트 없이 설정 변경, 기능 On/Off |
| 7주차 | **보안**          | 사용자 데이터 안전하게 보관             |

## ❓ 자주 묻는 질문

### Q. 이론만 배우나요?

**A.** 아니요! 매주 **이론 30분 + 실습 30~45분** 구성입니다.

- 실제로 테스트 코드 작성
- 실제로 CI/CD 파이프라인 구축
- 실제로 Remote Config로 앱 동작 제어

### Q. 어려울 것 같아요...

**A.** 걱정 마세요!

- 각 주제를 1시간~1시간 반에 소화할 수 있도록 **핵심만** 다룹니다
- 복잡한 설정은 **템플릿**을 제공합니다
- 모르는 건 **질문**하시면 됩니다

### 1. 실무 역량

- 테스트 → 품질 관리 능력

- Remote Config → 동적 앱 제어 능력
